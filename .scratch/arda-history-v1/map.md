# Wayfinder Map: arda_history v1 design spec

<!-- label: wayfinder:map -->

## Destination

A written **v1 design spec** for `arda_history` — an *emergent Middle-earth history simulator*. You watch history unfold tick-by-tick on a **2D graphical map of NW Middle-earth** (the v7 map), starting at **TA 2965** seeded accurately to canon, advancing **one year per tick**, open-ended. v1 models **characters & dynasties, factions & territory, war & battles, diplomacy & construction, Sauron's rise, and the One Ring as a tracked on-map artifact**, with dynamics that are **emergent but weighted toward canon** (canonicity is lightly tunable). The spec is the finish line — this effort produces **no game code** — and is ready to hand to `/to-spec` → `/to-tickets` → `/implement`.

## Notes

- **Domain:** Tolkien's Third Age / War of the Ring. Consult `/domain-modeling` to keep terms straight (faction/culture/realm, character/dynasty, the Ring, Sauron, hex/region).
- **Skills to consult:** `/grilling` + `/domain-modeling` (design decisions), `/research` (lore & tech facts), `/prototype` (spatial substrate, UI feel), `/codebase-design` (module shapes for the spec).
- **Standing preferences:**
  - Language: **Python**. Watch-only (no player control) in v1.
  - Geography is **fixed**. The **v7 Middle Earth map** (`references/Middle Earth v7.jpg`) is the canvas; the **Eriador TOR hex map** (`references/Eriador - TOR Hex Map.webp`) is a higher-detail reference for hexing/terrain.
  - History is **emergent with canonical pressures**; a **light canonicity parameter** ships in v1, deeper tuning later.
  - **Off-map peoples** (Harad, Rhûn, Khand) are modeled only as **abstract allies/providers** to Sauron — never fully simulated.
  - Plan, don't do: every ticket resolves a **decision**, not a slice of the build.

## Decisions so far

<!-- index: one line per closed ticket — gist + link to the ticket that holds the detail -->

- [Research: canonical TA 2965 state](issues/03-research-canonical-2965-state.md) — full seed compiled ([research file](research/canonical-2965-state.md)): rulers of every realm, Sauron openly back since 2951 (Mt Doom not yet erupting), the Ring quiescent with Bilbo, character ages, a not-yet-born list (Frodo/Boromir/Faramir/Éomer/Éowyn/Sam/Merry/Pippin all post-2965), off-map providers, settlement placements. Takeaway: the Bilbo→Frodo inheritance must be a *tendency*, not scripted (Frodo b. 2968).
- [Spatial substrate: how the map becomes a simulatable space](issues/01-spatial-substrate.md) — **two-layer hybrid** locked: a **named region graph** (~50–100 polygons, terrain + owner + adjacency) carries politics/territory, and a **location+route graph** (settlements/fords/passes joined by canonical roads & rivers) carries movement & battle siting — armies/Ring hop node→node. All geometry stored in **v7-image pixel coordinates** with one **px-per-mile** calibration from the scale bar; movement budgeted in **miles/year** (distance sets journey length in ticks, not tactics). Terrain **hand-authored** per region/route (no raster tracing). World is the **tightened WotR theatre** (Eriador→Erebor→Gondor→Mordor); far north & deep Harad are static backdrop; **Harad/Rhûn/Khand are single abstract provider nodes** at edge gateways feeding Sauron. Geometry is fixed *config*, not run state.
- [Core entity & tick model](issues/02-core-entity-tick-model.md) — the shared spine locked: a single **`World` of id-keyed typed dataclass records** (character, faction, region, location, route, army, the Ring, provider, event), all cross-references **ids not pointers**, dead entities **tombstoned** never deleted. A year advances via a **fixed ordered system pipeline** — aging/births/deaths → faction decisions → diplomacy → movement → war → construction/economy → Sauron+canon-pressure → salience/bookkeeping — each system `(world, rng) -> events`, one seeded RNG threaded throughout for reproducibility. Events are **state-authoritative + append-only log** (not event-sourced): systems mutate state then emit immutable dated `Event`s indexed by subject/faction/year/type; save = state snapshot + log. Ids are a single **monotonic, never-reused** integer space; config geometry gets deterministic ids at scenario load. Starts **TA 2965**.
- [Research: Python rendering stack](issues/04-research-python-rendering-stack.md) — recommends **PySide6/Qt** (`QGraphicsView` map + native docking panels/timeline/annals) over game frameworks, since this is an inspection-and-playback app not an action game; backup **pygame-ce + pygame_gui**. Core principle: **framework-agnostic seeded sim emitting snapshots + event stream**; SHA-256 string seed, canonical-JSON saves, never pickle/`hash()`. Briefcase for macOS packaging. ([research file](research/python-rendering-stack.md)) — a recommendation, locked at spec time.
- [Character & dynasty system](issues/05-character-dynasty-system.md) — characters are id-keyed records on a **data-tabled per-race lifecycle** (phase 1 rolls natural/disease death + births off the seeded RNG; violent death is phase 5's). A **`race` table** gives mortality/lifespan; **immortals skip the death roll** and Elves carry a **light fade / sail-West drive** (depart → tombstoned `departed`; Elf realms hold a `withdrawing` posture). **No Dynasty entity** — bloodlines are a query over kinship id-fields; realms use a **`succession_rule`** enum, and on a **failed line** try kin → elective fallback, else fragment/absorb into the strongest neighbor (zero-territory faction tombstoned, **dormant claim persists** → Arnor reachable). Behaviour from a small **trait vector** (`leadership/martial/ambition/loyalty`, mild heritability) + role enum; relationships = **kinship id-fields + a thin event-created sentiment layer** (rivalry/oath/friendship). Bilbo→Frodo stays a *tendency* at the 05↔09 seam.
- [Faction & territory system](issues/06-faction-territory-system.md) — a faction is one record tagged **`realm | culture | provider`**; concrete **TA 2965 roster** locked (Gondor/Rohan/Isengard/Mordor + **Dol Guldur distinct**, **Durin's Folk = Erebor+Iron Hills as one**, 4 separate Elf realms, Shire/Bree/Dunland/Havens as cultures, **landless Rangers with a thin "unclaimed North" claim**). Territory = **atomic region ownership**; "contested"/borders are **derived**; ownership flips by **conquering a region's `seat_location_id`** in phase 5. Faction AI = **weighted-utility rules** (no learning AI) in phase 2, with the **canonicity parameter as a global scalar weight**. **Providers = abstract gateway nodes** that **spawn real provider-flagged allied hosts** at their gateway (Corsairs raid coasts) — interceptable/routable, no interior, never conquered. New factions only via canon **restoration arcs**; extinction tombstones + keeps a dormant claim.
- [Chronicle & event surfacing in the UI](issues/11-chronicle-event-ui.md) — surfaced via a **virtualized `QListView` annals feed** (important-only default, filters on the four event indices, on-map pulses), a **graphics-hit-test inspection dock** (entity fields + its by-subject event timeline; clickable ids resolve even for tombstoned entities), and **snapshot-per-year playback** where **scrub restores snapshot[T] — never replays** (keyframe+replay is the at-scale fallback). Salience = a **deterministic absolute `importance` 0–100** scored at emission (type × subject-prominence × scale × canon-bump), immutable; UI may normalize to a top-N-per-year *view*. Prose = **template + seeded phrase-grammar into `Event.text`** — deterministic, offline, no deps; **LLM prose deferred** but the `text` field is the swappable seam.
- [World-state persistence: save/load + string seed](issues/12-persistence-save-load-seed.md) — a save is a **canonical-JSON snapshot** (state + event log) via `to_dict`/`from_dict` behind a **storage-backend seam** (event log can later move to SQLite without touching sim code); load = **rehydrate, not replay**. **Autosave every N years** at phase 8 + manual. Seed = **SHA-256(string) → `random.Random`**, one RNG threaded through; **`rng.getstate()` captured** so resume is bit-identical. A run is identified by **(scenario_id+version, canonicity_params, seed_str, code_version)** — scenario referenced not inlined — giving both "vary canonicity" and "vary seed" axes. Versioning = **provenance header + numbered migration chain**.
- [War & battle system](issues/07-war-battle-system.md) — war runs in **phase 5** on `Army` records: mustered from a region-derived manpower pool, moved as passengers on the route layer, fought via **strength-ratio + one bounded seeded-RNG roll** (resolves in-tick; **sieges persist across ticks**). Conquest = **hold a region's `seat_location_id`** → optional **raze** (→ `ruin`) by posture. Named-general death = a **rare post-battle scaled roll** → triggers 05 succession. **Canon pressure weights intents only, never battle dice.** Providers fight as ordinary hosts with **unit modifiers** (mûmakil/cavalry/aux); **Corsairs = coastal raids** (no seat seizure). Supply = light attrition. **07 is destructive/capture-only — all peacetime founding/rebuilding is 08's.**
- [Diplomacy & construction system](issues/08-diplomacy-construction-system.md) — diplomacy in **phase 3**: an **asymmetric per-faction `disposition` scalar** (sparse) + a **derived stance** (`alliance/neutrality/hostility/vassalage`) with pinned flags. **Wars formally start and end in phase 3** (07 only executes); peace/tribute/vassalage close them. Construction in **phase 6**, priced against a **lean single `treasury` scalar** (income = owned-region yields) — scarcity makes razing bite; population is a derived aggregate. **Vassalage = a directional overlord bond** (vassal musters for overlord, keeps succession, can rebel) — the mechanism for the **Reunited-Kingdom arc (bond, not faction-merge)** and the shape of **provider-pacts**. Marriage: **08 decides whether, 05 defines what it does.**
- [The One Ring artifact model](issues/09-one-ring-artifact-model.md) — a **single bespoke `Artifact`** with `bearer_id` **XOR** `location_id`; a **passenger** moving only with its bearer (no autonomous locomotion). Two integer scalars: **`corruption`** (per-bearer, trait-modulated, **attenuates on transfer** so ex-bearers stay marked; suppresses natural death = longevity) and **`pull`** (global, Sauron-facing). Possession changes via a fixed **transfer-mode enum** (inheritance biased along 05 kinship by canonicity = the Bilbo→Frodo *tendency*; theft/loss/found/war-capture/errand), with high `pull` raising loss/theft odds. **09 owns/computes `pull` in phase 7; ticket 10 consumes it** (searcher/Nazgûl + strength curve graduate to 10). Terminals: **destroyed** (gated on Mount Doom active ~3007+ → tombstoned), **Sauron reclaims**, **lying lost**; a non-Sauron claimant is a **transient**, not a 4th terminal. Seed: quiescent with Bilbo, both scalars low.
- [Sauron's rise & canonical-pressure model](issues/10-sauron-rise-canonical-pressure.md) — Sauron = the Mordor faction + a **`sauron_strength` scalar** computed each **phase 7** as **`canon_baseline(year) × canonicity + Σ emergent_deltas`** (arming since 2951, ramp to WotR, Orodruin active ~3007; gaining the Ring spikes it, defeats/Dol-Guldur-loss check it; at canonicity=0 the baseline flattens → fully emergent). Strength drives Mordor musters, provider commitment, Nazgûl activation, and the `pull`-rise weighting. Agents = **nine named immortal Nazgûl** (Witch-king lead) that act as **searchers** off 09's `pull`+location (10 reads, never writes the Ring) and as elite war-leaders. **Canon pressure = soft weighting only** — never fires events, never overrides dice — via a **single global canonicity 0–1 knob** (in `World.config`, saved separately) that nudges four forces: **Sauron's rise, the Ring's stirring, Free-Peoples alliances, character role-seeking**. Win = reclaim Ring or conquer the West; loss = Ring destroyed → Mordor dissolves via 06 extinction. **This closes the map** — the way to the v1 spec is clear.

## Not yet specified

<!-- in-scope fog: real questions we can't phrase sharply yet; graduate to tickets as the frontier advances -->

- **Aftermath modeling (largely resolved-by-continuation)** — with war/Ring/Sauron settled (07/09/10), a terminal outcome just **flips a world flag and the open-ended systems keep running** (Mordor dissolves via 06 extinction, Elves fade via 05, factions expand into the vacuum) — v1 needs **no special post-climax subsystem**. What remains fog is only long-tail *balancing/pacing* over centuries, tied to the performance-at-scale spike.
- **Magic & metaphysics depth beyond immortality & the Ring** — Maiar (Gandalf, the Balrog), palantíri, other artifacts, "power" as a modeled quantity. Elven immortality *is* in scope (ticket 05); the rest is fog.
- **Content-authoring breadth** — how much canonical detail we hand-author for every faction/settlement vs. generate. The build assets now committed (all *data*, not decisions): the **region/route/location dataset** in v7 pixel coordinates (~50–100 regions incl. each region's **`seat_location_id`** + **`base_yield`**, terrain/adjacency, canonical road/river routes); per-faction **seed values** (aggression/posture/disposition baselines); provider **`output` → combat-modifier profiles**; **battle/siege tuning** (fortification values, casualty/attrition curves, siege rates); the **treaty taxonomy** + disposition effects; **Ring corruption/pull coefficients**; and the chronicle's **per-event-type prose templates + phrase-grammar**. Only ticket 10's fields remain to freeze; then this is a single authoring pass, not a decision.
- **War-as-entity (deferred)** — whether a conflict becomes a first-class record (belligerents, war-goal, start/end year — a nameable "War of X" for the chronicle) or stays the per-pair at-war flag 07/08 use. Flagged jointly by 07 & 08; left as fog for v1.
- **Event-type catalog (spec-consolidation, not a decision)** — 02's `Event.type` enum was left open-ended and each system ticket added its own types (`birth/death/battle/siege/razing/conquest/founding/treaty/marriage/succession/ring_moved`/pull-changed/war-declaration/provider-pact/`departed`/`destroyed`/`sauron_grows`…). `/to-spec` should fold these into one canonical catalog, each with its structured `payload` shape + salience base-weight (11). A consolidation pass, not an open decision.
- **Performance at scale** — cost of an open-ended yearly sim over centuries across the whole map, *and* the growth of the append-only event log + per-year snapshot cache (drives whether/when 12's storage seam swaps JSON→SQLite, and whether 11 falls back to keyframe+replay). Likely a small research/prototype spike; revisit once the war/economy systems land.
- **Float-determinism policy** — 12's exact-resume guarantee requires the system tickets (05–10) to avoid cross-platform float branch-points (use integer/fixed-point where a comparison decides an outcome). A cross-cutting constraint to pin as each system ticket is worked, not a standalone decision.
- **Optional LLM prose enrichment** — a future, out-of-band way to enrich `Event.text` beyond templates (11 carved the seam). Deferred; if taken up it needs its own ticket with offline/reproducibility/packaging constraints.
